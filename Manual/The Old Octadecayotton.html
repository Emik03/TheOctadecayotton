On the Subject of The Octadecayotton

who'll be there when i'm gone?

The module's appearance is initially blank. Select the module to reveal a 9-dimensional cube represented in 512 spheres.

These spheres will start moving, resembling the rotations of a 9-dimensional cube. 3 rotations are shown, followed by a brief pause to indicate the end of its sequence.

Whenever D is mentioned, it refers to the amount of dimensions there are. Usually this is 9 (as the module's name implies), but can vary with mod settings.

Identifying Dimensions

  • There are 9 dimensions: X, Y, Z, U, V, W, R, S, and T. Each axis can either be positive, or negative.
  • Any positive axis points to the right (+X), up (+Z), and/or towards (+Y) the viewer when viewed from the front.
  • Any negative axis points to the left (-X), down (-Z), and/or away (-Y) the viewer when viewed from the front.
  • Any given sphere has 9 neighbours, which are all 1 of each of the 9 axes, either positively or negatively.
  • To the diagram to the right shows on example of a sphere's 9 neighbours as seen from the front. The unfilled spheres are neighbours of the filled sphere.
  • In reading order (left-to-right then top-to-bottom), the spheres are S, Z, V, W, U, T, R, Y (stacked on top of the filled sphere), and X.
  • In this example, going from the filled sphere to any unfilled is a positive axis, while going from an unfilled sphere to the filled sphere is a negative axis.

Identifying Rotations

  • Look at 2 spheres that are neighbours (note down their initial axis that connects them together) and watch them transform.
  • If these 2 spheres are now neighboured from a different axis, that implies a rotation. Record the new axis that they transformed into, and repeat this process starting with the new axis until the first initial axis is reached.
  • For any negative initial axis, invert both the initial and new axis. (positive -> negative, negative -> positive)

Identifying Rotations (...continued)

  • When the first initial axis has been reached, start from the bottom of the list, and working your way up; append all the new axes to a separate list.
  • The resulting list is called a subrotation. In a given rotation there can be multiple simulatenous subrotations happening at the same time.

Primary Values

  • All rotations and subrotations have a primary value. Within each subrotation, create a list of every possible pair of axes from their neighbours. (including the last and first axis as a pair*)
  • If the subrotation contains exactly 1 axis, ignore the above rule and make only 1 pair, consisting of the axis, and the positive version of that axis.
  • For each pair, get the value from the table, using the first letter as the column and the second letter as the row.

* Even with 2 axes, this rule still applies. For example, subrotation +R-T gives -RT and -TR.

  • If 1 of the 2 axes are negative, multiply the pair's result with -1.
  • The sum of all pairs is the primary value of the subrotation. The sum of all subrotations in 1 rotation is the primary value of that rotation.
  • Later in this manual, whenever p1, p2, or p3 is mentioned, it refers to the primary value of the 1st, 2nd, and 3rd rotation respectively.

The Second Rotation

  • The second rotation will always be 1 subrotation that includes all axes.
  • Get an axis from "Number <-> Axis" with the input being p2 modulo* D.
  • Reorganise the subrotation such that the received axis is the first element of the list.
  • Since every axis is positive or negative, this means that the rotation can also point to 1 sphere that shares all negative/positive values of the rotation. This is called the starting sphere.

* Modulo is to add/subtract the right number until the left is non-negative, and less than the right.

The Binary Logic Gate

  • To get the output of a given sphere, reorder the axes to be the starting sphere, and convert it to gray code by appending a 1 on positions where the starting sphere and the sphere to calculate have the same positive/negative value, and 0 otherwise. Convert it to binary:
    1. The first binary digit will match the first digit of the gray code.
    2. The next digit is a 1 if the sum of the previous digit of the gray code and the current position's binary code is exactly 1. Otherwise it's a 0.
    3. Repeat step 2 until D digits are obtained. This is the binary code.
  • Convert the binary code to decimal by adding up all the numbers on the below table's "Decimal" only if the corresponding position in "Binary" is 1.
Binary to Decimal
Binary (Xth from the right) 9st 8nd 7rd 6th 5th 4th 3rd 2nd 1st
Decimal 256 128 64 32 16 8 4 2 1
  • Add the decimal number with p2, then take modulo 9. Starting from 0 and on the leftmost axis of the second rotation's subrotation, go right 1 axis until the number obtained is reached.
  • The current axis' positivity/negativity is the output.

The Third Rotation

  • The third rotation will always includes all axes across subrotations.
  • Each subrotation has a secondary value. Construct a function using the following rules, then calculate the function to get the secondary value. All conditions must be followed, even after a condition is true.
    1. Start with the primary value multipled by -1 of the subrotation.
    2. If the subrotation contains exactly 1 of X, Y, and Z: Add D2.
    3. If the subrotation is 3 axes long: Add the sum of those axes by converting each one with "Number <-> Axis".
    4. If the subrotation has equal amounts of positive and negative axes: Add the amount of axes of the subrotation.
    5. If the subrotation connects any of +X, +W and +R: Subtract 1.
    6. If the subrotation connects any of +Y, +U and +T: Subtract 10.
    7. If the subrotation connects any of +Z, +V and +S: Subtract 100.
    8. If the amount of axes of the subrotation is unique within the third rotation: Add D plus 2.
    9. If the amount of negative axes in the subrotation is equal to the amount of subrotations in the third rotation: Add the amount of subrotations multiplied by D.
    10. If the first rotation's combined subrotations points to a sphere that returns positive (+) on the logic gate: Add all axes not included in the first rotation from "Number <-> Axis".
    11. If the starting sphere from the second rotation points to a sphere that returns positive (+) on the logic gate: Add the last axis's value from "Number <-> Axis".
    12. If the third rotation's combined subrotations points to a sphere that returns positive (+) on the logic gate: Add all axes in the third rotation who are alone in a subrotation from "Number <-> Axis".

Variables

  • D = the amount of dimensions there are.
  • px = the primary value of x.
  • sx = the secondary value of x.
  • ax = the amount of axes x has.
  • tx = the amount of conditions used for function x, including the first.
  • fx(y) = the function that was used to get sx, replacing primary values with y.
